# CSS Methodologies Notes

## Why CSS Methodologies Matter

CSS is globally scoped by default, which leads to several challenges:

- The cascade can affect unintended elements.
- Global styles may conflict with component-specific styles.
- Naming collisions are common.
- High specificity makes maintenance harder.
- Large stylesheets become difficult to navigate.

## Common Problems

- Unpredictable style overrides
- Poor organization
- Difficulty scaling styles for larger projects
- Code that is hard to refactor

## Benefits of a Methodology

- Better structure and consistency
- Reduced naming conflicts
- Easier team collaboration
- Clearer component and page boundaries
- More predictable styling behavior

## Methodology Examples

- BEM (Block Element Modifier)
- OOCSS (Object-Oriented CSS)
- SMACSS (Scalable and Modular Architecture for CSS)
- Atomic CSS / Utility-first CSS

Each methodology helps keep styles durable and maintainable in larger projects.
